# AI Test Case Generator 📋
A specialized productivity tool designed for Quality Assurance Engineers and Developers. This application leverages the power of Gemini AI to transform software requirements, feature descriptions, or user stories into professional, industry-standard test cases in seconds.

🔗 **Live Demo**
Try it now without any installation: [AI Test Case Generator on AI Studio](https://ai.studio/apps/b4c304e0-8d7c-4ac8-b3de-3e04b6dfb186?fullscreenApplet=true)

🎯 **Why This Tool?**
In a fast-paced software development lifecycle, writing comprehensive test cases is critical but often tedious. This tool bridges the gap between "understanding requirements" and "verifying them" by automating the test design process, ensuring that every feature is covered by clear, actionable, and professional test scenarios.

🚀 **Features**
- **Intelligent QA Analysis**: Uses Gemini 3.1 Pro to parse complex requirements and extract critical testing components.
- **Professional Structuring**: Automatically generates:
    - **Unique ID & Title**: Scannable and organized.
    - **Preconditions**: Clearly defined environment setup.
    - **Sequential Steps**: Easy-to-follow instructions for execution.
    - **Expected Results**: Precise success criteria for each scenario.
- **Smart Metadata**: AI-estimated Priority (High/Medium/Low) and Test Type (Functional, UI, Security, etc.).
- **Efficiency First**: Multi-format export options including PDF, Word (.doc), Text, and JSON for seamless integration with Jira, TestRail, or Zephyr.
- **Modern QA Interface**: A clean, focused UI built with React and Tailwind CSS, featuring smooth animations and a responsive layout.

🛠️ **Tech Stack**
- **Frontend**: React 19, TypeScript, Tailwind CSS
- **AI Engine**: Google Gemini API (@google/genai)
- **Animations**: Motion (Framer Motion)
- **Icons**: Lucide React
- **Export**: jsPDF, Document Processing

🏁 **Getting Started**
1. **Clone the repository**:
   `git clone https://github.com/SahilBQA/ai-test-case-generator.git`
2. **Install dependencies**:
   `npm install`
3. **Configure Environment**: Create a `.env` file in the root directory:
   `GEMINI_API_KEY=your_gemini_api_key_here`
4. **Launch the App**:
   `npm run dev`
